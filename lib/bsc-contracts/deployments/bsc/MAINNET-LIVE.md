## VPS install — quick path

```bash
ssh root@93.127.213.192
cd /opt/zebvix && git pull
# 1-of-1 bootstrap — only provision env file for signer-1
sudo VALIDATOR_COUNT=1 bash lib/bridge-deployment/install-vps.sh
```

The installer auto-generates a shared `SIGNER_AUTH_TOKEN` and injects it into
`relayer.env` + `signer-1.env`. You only need to fill the secrets below.

## Migration path (when you're ready to harden)

1. **Deploy Gnosis Safe on BSC.** 3-of-5 hardware wallets recommended.
2. **Generate fresh validator keys offline:** `N=4 bash lib/bridge-deployment/genkeys.sh > validators.json` (we already have validator #1).
3. **From deployer EOA, call:**
   - `ZebvixBridge.transferOwnership(SAFE_ADDRESS)`
   - `WrappedZBX.grantRole(DEFAULT_ADMIN_ROLE, SAFE_ADDRESS)`
   - `WrappedZBX.revokeRole(DEFAULT_ADMIN_ROLE, deployer)`
4. **From Safe, call:**
   - `ZebvixBridge.addValidator(0xVAL2)` × 4 (for the 4 new validators)
   - `ZebvixBridge.setThreshold(3)` (raises to 3-of-5)
5. **Add 4 more signer instances** on the VPS or separate hosts (zbx-signer@2..5).
6. Update `SIGNER_ENDPOINTS` in `/etc/zbx-bridge/relayer.env` and `systemctl restart zbx-relayer`.

No data migration, no funds at risk during this transition — all changes are owner-gated on-chain calls.
